# ScholarAgent 产品需求文档（PRD）

> 版本：v1.1 | 更新日期：2026-06-25

---

## 一、背景与目标

### 1.1 问题陈述

学术研究者在文献调研过程中面临三类痛点：

1. **检索分散**：arXiv、Semantic Scholar 等平台各自独立，难以在一次对话中完成检索、筛选与对比。
2. **PDF 难消化**：上传的论文难以快速提炼要点，无法针对文档内容连续追问。
3. **引用繁琐**：从检索结果到 BibTeX 引用格式需要手工整理。

### 1.2 产品愿景

**ScholarAgent** 是一个基于 ReAct 推理框架的智能学术研究辅助工具，通过透明可观测的多步推理，为用户提供端到端的学术问答、文献检索与知识库问答体验。

### 1.3 成功指标

| 指标 | 目标值 | 测量方式 |
| ---- | ------ | -------- |
| 端到端问答延迟 | P95 < 30s（Phase 1 CLI） | Prometheus `llm_request_duration_seconds` + 端到端计时 |
| 工具调用成功率 | > 95% | `agent_tool_calls_total{status="success"}` / 总量 |
| 思考链完整展示率 | 100%（Web 端） | 每次问答的 thought/action/observation/answer 四类事件均到达前端 |

## 三、功能需求

| -- | ---- | ---- | ------ | ----- |

## 四、非功能需求

### 4.1 性能

| 项 | 要求 |
| -- | ---- |
| 首 token SSE 延迟 | P95 < 3s（Web 端，不含工具执行） |
| PDF 解析（10MB 内） | Phase 2 目标：上传后 2 分钟内可 RAG 查询 |
| ReAct 最大步数 | 默认 5 步，超限返回 `error` 事件 |

### 4.2 可用性与降级

- 主力 LLM（`deepseek-v4-flash`）调用失败时，自动降级至 MiMo V2.5 Pro，再降级至 MiniMax-M3。
- Function Calling 解析失败时，回退 JSON Mode。
- 单微服务实例故障时，K8s 滚动更新保证服务连续性（Phase 3）。

### 4.3 成本

- Agent 循环默认使用 DeepSeek V4-Flash **non-thinking 模式**（`extra_body: {"thinking": {"type": "disabled"}}`），降低延迟与 Token 费用。
- 最终综合回答可选用 thinking 模式以提升质量（可配置）。

### 4.4 安全

- 所有 LLM / OCR API Key 仅存服务端环境变量，禁止写入前端或版本库。
- PDF 向量数据默认存储于本地 Qdrant；云端 OCR（百度）为可选路径，需用户知情。
- MVP 不实现完整用户认证；使用匿名 `device_id` Cookie 区分会话（见 D-009）。

## 五、范围边界（Out of Scope）

以下功能明确不在当前三阶段范围内：

- 用户注册 / 登录 / OAuth 系统
- 论文全文付费获取、机构 VPN 代理接入
- 多语言 UI（MVP 仅中文界面，学术内容支持英文）
- 协作编辑、会话分享、权限管理
- 移动端原生 App

## 六、竞品与差异化

| 产品 | 优势 | ScholarAgent 差异化 |
| ---- | ---- | ------------------- |
| Consensus | 学术搜索体验成熟 | 自建 ReAct 工具编排，推理链完全可观测 |
| Elicit | 结构化文献分析 | 可自部署微服务，支持 PDF 私有 RAG |
| ChatPDF | PDF 问答简单直接 | 结合公开论文检索 + RAG + 引用生成的一体化 Agent |

**核心差异化**：可观测 ReAct 思考链 + 自建工具编排 + Go 微服务可部署示范。

## 七、附录：风险登记

| 风险 | 影响 | 概率 | 缓解措施 |
| ---- | ---- | ---- | -------- |
| 三微服务 + K8s 三周工期紧 | 延期 | 中 | Phase 1 先打通 gRPC 最小路径；可观测性推迟至 Phase 3 |
| 百度 OCR MCP 配额 / 稳定性 | PDF 解析失败 | 中 | PaddleOCR 本地备轨；工具层抽象 OCR Provider |
| Semantic Scholar API 限流 | 摘要获取失败 | 中 | 主用 arXiv API，Semantic Scholar 作 fallback |
| DeepSeek thinking 默认增加延迟 | 用户体验差 | 高 | Agent 循环强制 non-thinking（见 §4.3） |
| `deepseek-chat` legacy ID 退役（2026-07-24） | 集成中断 | 低 | 统一使用 `deepseek-v4-flash` 模型 ID |
