# ScholarAgent 里程碑与验收标准

> 版本：v1.1 | 更新日期：2026-06-25

本文档定义三阶段交付的 **Definition of Done（DoD）**，每项任务关联 [产品需求.md](./产品需求.md) 中的功能 ID。

## Phase 1 — MVP（1 周）

目标：打通「提问 → ReAct 推理 → 工具调用 → 回答」最小闭环，三微服务可本地启动。

| 任务 | 关联功能 | 验收标准 | 状态 |
| ---- | -------- | -------- | ---- |
| Agent Core ReAct 循环 | F-01 | CLI 输入问题，输出 ≥1 次 tool call + 最终 `answer` 事件；`maxSteps=5` 超限时返回 `error` 事件 | [x] |
| DeepSeek 集成 | F-01 | `deepseek-v4-flash` Function Calling 调用成功；主模型失败时自动切换 MiMo | [x] |
| search_papers 工具 | F-02 | 对 `"attention mechanism"` 返回 ≥5 条结构化结果（标题、作者、年份） | [x] |
| get_abstract 工具 | F-02 | 对已知 arXiv ID 返回完整摘要文本 | [x] |
| Redis 会话管理 | F-03 | 服务重启后会话历史可恢复；TTL 7 天可通过环境变量配置 | [x] |
| 微服务骨架 | — | `gateway` + `agent-core` + `tool-service` 通过 `docker-compose up` 可启动；gRPC 通信正常 | [x] |
| CLI 交互测试 | F-01, F-02, F-03 | 端到端 CLI 脚本连续 3 次问答无崩溃，会话上下文可延续 | [x] |

### Phase 1 完成判定

- 上表全部 checkbox 勾选
- 成功指标（PRD §1.3）：工具调用成功率在 CLI 手动测试 10 次中 ≥9 次成功

## Phase 2 — 增强（1 周）

目标：RAG 知识库、PDF 异步解析、Web 前端与容器化交付。

| 任务 | 关联功能 | 验收标准 | 状态 |
| ---- | -------- | -------- | ---- |
| Qdrant RAG 模块 | F-05 | `rag_query` 对已入库文档返回 Top-5 相关片段；人工抽测 3 组问答全部命中预期段落 | [x] |
| PDF 解析（MCP OCR） | F-05 | 10MB 内 PDF 上传后 2 分钟内 `job` 状态变为 `completed` | [x] |
| RabbitMQ 异步任务 | F-05 | `parse_pdf` 发布消息 → `pdf-worker` 消费 → Qdrant 入库，全链路可追踪 | [x] |
| generate_citation | F-06 | 对已知 paper_id 输出合法 BibTeX 条目 | [x] |
| Web 聊天界面 | F-04 | 思考链四类事件（thought/action/observation/answer）均正确渲染；支持新会话与历史加载 | [x] |
| PDF 上传 UI | F-04, F-05 | 前端上传 PDF 并展示解析进度；完成后可在同会话 RAG 问答 | [x] |
| Docker 容器化 | — | 全部服务 + Redis + Qdrant + RabbitMQ 一键 `docker-compose up` | [x] |

### Phase 2 完成判定

- 上表全部 checkbox 勾选
- Web 端 POST `/api/v1/chat/stream` SSE 流式正常，首 token P95 < 3s（本地网络）
