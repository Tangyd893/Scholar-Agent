## 一、系统架构

### 1.1 微服务拓扑

```
┌─────────────────────────────────────────────────────────────────┐
│                  前端 (React + TailwindCSS)                     │
│              Agent 思考链可视化 + PDF 上传                         │
└───────────────────────────┬─────────────────────────────────────┘
                            │ HTTP / SSE (POST)
┌───────────────────────────▼─────────────────────────────────────┐
│              Gateway (Go + Gin)                                 │
│  - 静态资源 / REST API / SSE 流式推送 / 会话 Cookie 管理          │
└───────┬──────────────────────────────┬──────────────────────────┘
        │ gRPC                         │ Redis
┌───────▼──────────────┐    ┌──────────▼──────────────────────┐
│   Agent Core (Go)    │    │    Session Manager (Redis)      │
│  - ReAct + FC        │    │  - 会话元数据 / 消息历史 / 推理链   │
│  - LLM Gateway       │    └─────────────────────────────────┘
└───────┬──────────────┘
        │ gRPC
┌───────▼─────────────────────────────────────────────────────────┐
│                   Tool Service (Go)                             │
├─────────────────────────────────────────────────────────────────┤
│  search_papers │ get_abstract │ rag_query │ generate_citation  │
│  (arXiv API)   │ (Semantic)   │ (Qdrant)  │ (BibTeX)           │
└───────┬──────────────────────────────┬──────────────────────────┘
        │ MCP                          │ publish
┌───────▼─────────┐           ┌────────▼────────┐
│  OCR MCP Server │           │    RabbitMQ     │
│  (百度/Paddle)   │           └────────┬────────┘
└─────────────────┘                    │ consume
                              ┌────────▼────────┐
                              │   PDF Worker    │
                              │ OCR+分块+Embed  │
                              └────────┬────────┘
                                       │ write
                              ┌────────▼────────┐
                              │     Qdrant      │
                              └─────────────────┘
```

### 1.2 核心数据流（一次完整问答）

```
用户输入 → Gateway (POST /api/v1/chat/stream)
    → gRPC AgentCore.Run(sessionID, query)
    → 加载 Redis 会话历史
    → LLM (deepseek-v4-flash, non-thinking) + Tools
    → gRPC ToolService.Execute(toolName, args)
    → Observation 回注消息历史
    → 循环直至 answer 或 maxSteps
    → Gateway SSE 推送 StepEvent 至前端
```

## 二、服务边界与 gRPC 接口

### 2.1 Agent Core Service

```protobuf
service AgentCore {
  rpc Run(RunRequest) returns (stream StepEvent);
}

message RunRequest {
  string session_id = 1;
  string query = 2;
}

message StepEvent {
  string type = 1;      // thought | action | observation | answer | error
  string content = 2;
  int32 step = 3;

}
```

### 2.2 Tool Service

```protobuf
service ToolService {
  rpc Execute(ExecuteRequest) returns (ExecuteResponse);
  rpc IngestPDF(IngestRequest) returns (IngestResponse);
}

message ExecuteRequest {
  string tool_name = 1;
  string arguments_json = 2;
}

message ExecuteResponse {
  string result = 1;
  string error = 2;
}

message IngestRequest {
  string file_id = 1;
  string session_id = 2;
}

message IngestResponse {
  string job_id = 1;
}
```

## 三、核心模块设计

### 3.1 LLM Gateway

**职责**：封装不同 LLM 提供商的 API 调用，支持降级与故障转移。

```go
type LLMClient interface {
    Chat(ctx context.Context, req *ChatRequest) (*ChatResponse, error)
    ChatStream(ctx context.Context, req *ChatRequest) (<-chan StreamChunk, error)
}

type DeepSeekProvider struct {
    client *openai.Client
    model  string // "deepseek-v4-flash"
}
```

**关键设计**：

- 统一 OpenAI 兼容接口（DeepSeek、MiMo、MiniMax 均支持）
- MiniMax 采用 OpenAI 兼容格式以保持一致性
- 调用失败时按 DeepSeek → MiMo → MiniMax 顺序自动切换
- Agent 循环默认 **non-thinking**：`extra_body: {"thinking": {"type": "disabled"}}`
- 模型 ID 使用小写 `deepseek-v4-flash`；legacy `deepseek-chat` 将于 2026-07-24 退役

**DeepSeek-V4-Flash 选型依据**：

- 284B 总参数 / 13B 激活（MoE），推理成本低
- 100 万 token 上下文，适合长论文
- 原生 Function Calling，与 ReAct 设计匹配

### 3.2 Agent Core（ReAct 推理引擎）

```go
type Agent struct {
    llm       LLMClient
    toolSvc   ToolServiceClient  // gRPC 调用 Tool Service
    memory    MemoryStore
    maxSteps  int  // 默认 5
}

type StepEvent struct {
    Type    string // thought | action | observation | answer | error
    Content string
    Step    int
}
```

**ReAct 循环**（接口级描述）：

1. 从 Redis 加载 `session_id` 对应历史消息
2. 循环最多 `maxSteps` 次：
   - 调用 LLM（带工具定义，non-thinking 模式）
   - 无 Tool Call → 推送 `answer` 事件并结束
   - 有 Tool Call → 推送 `thought` / `action`，gRPC 调用 `ToolService.Execute`，推送 `observation`
   - 将工具结果追加到消息历史，保存推理链至 Redis
3. 达到 `maxSteps` → 推送 `error` 事件

**降级策略**：Function Calling 解析失败时，回退 JSON Mode。

### 3.3 工具服务（Tool Service）

| 工具名 | 功能 | 实现 | 输入参数 |
| ------ | ---- | ---- | -------- |
| `search_papers` | 关键词搜索论文 | arXiv API（主）/ Semantic Scholar（备） | `query: string` |
| `get_abstract` | 获取论文摘要 | 按 paper_id 查询 | `paper_id: string` |
| `rag_query` | 查询本地知识库 | Qdrant 向量检索 | `query: string`, `top_k: int` |
| `generate_citation` | 生成引用 | BibTeX 模板 | `paper_id: string`, `format: string` |
| `parse_pdf` | 解析 PDF | 发布 RabbitMQ 任务 | `file_id: string` |

`search_papers` 注册示例（参考 agent-sdk-go 模式）：

```go
searchPapers := tool.NewFunctionTool(
    "search_papers",
    "按关键词搜索学术论文，返回论文列表（含标题、作者、摘要、年份）",
    func(ctx context.Context, params map[string]interface{}) (interface{}, error) {
        query := params["query"].(string)
        return arXivClient.Search(query)
    },
).WithSchema(/* JSON Schema */)
```

### 3.4 MCP 集成（OCR/PDF）

**定位**：仅 PDF 解析通过 MCP 协议调用 OCR，其余工具为普通 Go 实现。

**双轨策略**：

1. **百度 OCR MCP**（主）：配置即用
2. **PaddleOCR + MCP**（备）：本地私有化部署

```json
{
  "mcpServers": {
    "ocr_document": {
      "url": "https://aip.baidubce.com/mcp/document/sse?Authorization=Bearer%20<API Key>"
    }
  }
}
```

工具层抽象 `OCRProvider` 接口，便于主备切换。

### 3.5 RAG 知识库（Qdrant）

**数据模型**：

```go
type PaperChunk struct {
    ID        string                 `json:"id"`
    PaperID   string                 `json:"paper_id"`
    Title     string                 `json:"title"`
    Content   string                 `json:"content"`
    Embedding []float32              `json:"embedding"`
    Metadata  map[string]interface{} `json:"metadata"`
}
```

**Embedding 模型**（D-010）：`text-embedding-3-small`（OpenAI 兼容 API）；维度 1536。

**分块策略**：

- 块大小：512 token
- 重叠：50 token
- Collection 名称：`papers`

**检索流程**：

1. 用户查询 → Embedding 模型生成向量
2. Qdrant 相似度检索（Top-K=5，默认）
3. 结果作为上下文注入 LLM

**入库触发**：PDF Worker 完成 OCR + 分块 + Embedding 后批量写入 Qdrant。

### 3.6 RabbitMQ 异步 PDF 解析

```
parse_pdf / POST /api/v1/papers/upload
    → Tool Service 存储文件，发布消息 queue: pdf.parse
    → PDF Worker 消费：
        1. MCP OCR 提取文本
        2. 分块（512 token / 50 overlap）
        3. Embedding 向量化
        4. 写入 Qdrant collection papers
        5. 更新 job 状态 → completed
    → 前端轮询 GET /api/v1/jobs/{id} 或接收 SSE job_complete
```

消息体示例：

```json
{
  "job_id": "job_xyz789",
  "file_id": "file_abc",
  "session_id": "sess_abc123"
}
```

### 3.7 会话管理（Redis）

| Key 模式 | 类型 | 内容 | TTL |
| -------- | ---- | ---- | --- |
| `session:{id}:meta` | String(JSON) | user_id(device_id), created_at, title | 7 天 |
| `session:{id}:messages` | List | 消息历史（JSON） | 7 天 |
| `session:{id}:steps` | List | Agent 推理链 | 7 天 |
| `user:{id}:sessions` | Set | 会话 ID 列表 | 永久 |

```go
type SessionManager struct {
    redis *redis.Client
}

func (m *SessionManager) Create(userID string) (string, error)
func (m *SessionManager) GetHistory(sessionID string, lastN int) ([]Message, error)
func (m *SessionManager) Append(sessionID string, msg Message) error
func (m *SessionManager) Touch(sessionID string) error
```

MVP 身份：`device_id` Cookie / `X-Device-ID` Header（D-009）。

## 五、部署方案

### 5.1 Docker 容器化

每个服务独立 Dockerfile；`deploy/docker-compose.yml` 编排全部依赖：

- 应用服务：gateway、agent-core、tool-service、pdf-worker
- 基础设施：redis、qdrant、rabbitmq
- 可观测性：prometheus、grafana（Phase 3）

### 5.2 Kubernetes（Minikube）

- 每个服务 2 副本，滚动更新
- gRPC 服务通过 K8s Service 发现（`agent-core:50051`、`tool-service:50052`）
- Readiness / Liveness Probe
- HPA：CPU 70% 触发扩容

```yaml
resources:
  requests:
    memory: "256Mi"
    cpu: "250m"
  limits:
    memory: "512Mi"
    cpu: "500m"
```

### 5.3 本地开发 docker-compose 片段

```yaml
services:
  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  qdrant:
    image: qdrant/qdrant:latest
    ports: ["6333:6333"]
    volumes: ["./qdrant_storage:/qdrant/storage"]

  rabbitmq:
    image: rabbitmq:3-management-alpine
    ports: ["5672:5672", "15672:15672"]
```

完整配置见 `deploy/docker-compose.yml`（Phase 2 实现时创建）。

## 六、技术决策记录（ADR）

| 编号 | 决策项 | 最终选择 | 备选方案 |
| ---- | ------ | -------- | -------- |
| D-001 | 主力 LLM | DeepSeek-V4-Flash (`deepseek-v4-flash`) | MiniMax-M3 |
| D-002 | ReAct 实现 | Function Calling + JSON Mode 兜底 | 纯文本 ReAct |
| D-003 | 向量数据库 | Qdrant | Chroma |
| D-004 | 会话存储 | Redis | 内存 |
| D-005 | 异步任务 | RabbitMQ | 直接执行 |
| D-006 | OCR 集成 | 百度 OCR MCP（主）+ PaddleOCR MCP（备） | 单一 OCR |
| D-007 | 可观测性 | Prometheus + Grafana | 日志系统 |
| D-008 | 前端框架 | React + Vite | 纯 HTML |
| D-009 | MVP 身份方案 | 匿名 `device_id` Cookie / Header | 完整 OAuth |
| D-010 | Embedding 模型 | `text-embedding-3-small` | DeepSeek embedding |
| D-011 | SSE 传输 | POST + `text/event-stream` | GET query 参数 |
